# PPT智能生成与文档库管理系统 - 需求文档

## 1. 项目概述

### 1.1 项目背景
本项目旨在开发一个基于Web的智能PPT生成系统，允许用户建立本地的PPT文档库，并通过自然语言对话的方式，从已有PPT中抽取相关页面和内容，自动生成新的PPT文档。系统支持多轮对话修改，并确保新PPT的风格一致性。

### 1.2 项目目标
- 建立高效的PPT文档管理和检索系统
- 实现基于AI的智能PPT内容抽取和生成
- 提供自然语言交互界面，简化PPT创建流程
- 支持多轮对话迭代优化PPT内容
- 保证生成PPT的风格统一性和专业性

### 1.3 目标用户
- 需要频繁创建PPT的商务人士
- 培训机构讲师
- 市场营销人员
- 教育工作者

## 2. 功能需求

### 2.1 核心功能模块

#### 2.1.1 PPT文档库管理

**功能描述**：用户可以上传、管理、检索本地的PPT文档。系统对每一页内容进行深度解析并向量化存储，用于后续的语义检索和页面复用

**详细需求**：
- 支持上传PPT文件（支持格式：PPTX, PPT），**无文件大小限制**
  - 大文件采用分片上传机制
  - 支持断点续传
  - 上传进度实时显示
- **页面级解析**：自动将PPT按页拆分，提取每页的完整内容
  - 文本内容（标题、正文、列表等）
  - 图片及其OCR识别内容
  - 图表数据和类型
  - 布局结构信息
  - 样式信息（配色、字体等）
- **向量化存储**：将每页内容转化为向量嵌入存储
  - 文本内容使用语义模型（如m3e-base）生成向量
  - 图片内容使用CLIP等多模态模型生成向量
  - 存储到向量数据库（Chroma/Milvus）
- 支持文档分类管理（文件夹/标签）
- 文档元数据管理（标题、作者、创建时间、页数等）
- **页面级语义检索**：支持基于自然语言描述检索相关页面
- 文档预览功能（支持单页预览）
- 文档删除和编辑
- 批量导入导出

**验收标准**：
- 支持上传任意大小的PPT文件（无大小限制）
- 页面解析准确率 > 95%
- 页面级语义检索响应时间 < 2秒
- 支持并发上传至少10个文件

#### 2.1.2 PPT大纲设计

**功能描述**：在页面组装之前，用户需要先设计PPT的整体大纲结构。系统提供两种大纲生成方式：智能生成和向导式生成，帮助用户快速构建清晰的PPT框架

**详细需求**：

##### 方式一：智能生成（一键生成）

用户输入一段描述性文字，系统自动解析并生成完整的PPT大纲结构

- **输入区域**：提供大文本输入框，支持用户输入PPT需求描述
- **示例提示**：文本框内显示占位提示，引导用户输入关键信息
- **AI解析**：系统使用AI模型解析用户输入，自动提取并生成大纲
- **结果编辑**：生成的大纲支持手动调整和修改

**输入框示例提示**：
```

  - 第四章：案例展示（展示成功客户案例）
  - 第五章：商务合作（说明合作方式和联系方式）
```

##### 方式二：向导式生成（分步引导）

通过分步向导引导用户逐步完善PPT大纲，适合不确定整体结构的用户

**步骤1：PPT主题设定**
- **PPT标题**：输入PPT的主标题（必填）
- **制作目标**：描述制作这个PPT的主要目的（必填）
- **目标受众**：选择或输入目标观众群体（可选）
- **预计时长**：选择演示时长（5分钟/10分钟/15分钟/30分钟/自定义）

**步骤2：章节规划**
- **添加章节**：点击"+"按钮添加新章节
- **章节标题**：为每个章节设置标题（必填）
- **预估页数**：设置每个章节的预期页数（1-5页）
- **章节排序**：支持拖拽调整章节顺序
- **删除章节**：支持删除不需要的章节
- **快捷模板**：提供常用章节结构模板（如：产品介绍、工作汇报、培训课件等）

**步骤3：章节内容摘要**
- **内容描述**：为每个章节填写内容摘要（必填）
- **关键词标签**：添加章节相关的关键词，提高检索准确性
- **AI建议**：系统根据章节标题给出内容建议
- **参考文档**：可选择指定从哪些文档中检索页面

**步骤4：风格选择（可选）**
- **预设风格**：选择系统提供的预设风格模板
- **自定义风格**：上传参考PPT提取风格
- **跳过此步**：不设置风格，保留原始页面样式

#### 2.1.3 PPT页面拼凑组装

**功能描述**：基于已设计好的PPT大纲，系统从文档库中检索相关页面，以"页面复制"方式拼凑组装新PPT，不做内容生成和修改，仅做页面筛选和组合

**详细需求**：
- **大纲关联**：组装页面与大纲设计模块关联，自动获取章节结构和内容摘要
- **智能检索**：根据大纲中的章节内容摘要，从文档库中语义检索最相关的页面
- **备选页面展示**：每个章节位置展示多个（3-5个）备选页面卡片
  - 卡片展示页面缩略图预览
  - 显示页面来源文档和原始页码
  - 显示与当前章节需求的匹配度分数
  - 支持左右滑动或网格浏览备选页面
- **页面选择**：用户从备选卡片中选择最合适的页面加入新PPT
  - 点击卡片选中/取消选中
  - 支持拖拽调整页面顺序
  - 支持删除已选页面重新选择
- **纯复制模式**：选中页面直接复制，不做任何内容修改或AI生成
- **页面替换**：针对已选中的页面，可基于该页面内容重新检索相似页面进行替换
  - 点击"替换"按钮，系统根据该页面内容特征检索3个最相似的备选页面
  - 以卡片形式展示备选页面供用户选择
  - 选择新页面后替换原页面位置
- **草稿保存**：支持随时保存当前组装进度为草稿

**使用流程**：
1. 完成PPT大纲设计（智能生成或向导式生成）
2. 进入页面组装界面，系统自动根据大纲章节检索相关页面
3. 每个章节位置展示多个备选页面卡片
4. 用户浏览并选择最合适的页面
5. 如需替换某页，点击"替换"按钮，系统展示3个相似页面供选择

#### 2.1.5 PPT精修（AI对话编辑）

**功能描述**：PPT组装完成后，用户可以通过对话交互方式对每一页进行精细化编辑修改

- **对话式编辑交互**：
  - 用户选中页面中的特定元素（文本框、表格、图片等）
  - 通过自然语言对话发起修改指令
  - AI理解意图并执行具体的编辑操作
  - 实时预览修改效果

- **支持的修改类型**：
  - **文本编辑**：修改标题、正文内容，润色表述，调整字体/颜色/大小
  - **表格编辑**：修改单元格内容，添加/删除行列，调整格式
  - **图片操作**：替换图片，调整位置和大小
  - **元素增删**：添加新文本框、形状，删除不需要的元素
  - **布局调整**：移动元素位置，调整层级关系
  - **AI生成**：根据上下文生成标题、要点、段落内容

- **智能辅助功能**：
  - **快捷操作**：根据选中元素类型显示常用操作按钮
  - **智能建议**：自动检测页面问题并给出优化建议
  - **一键优化**：支持一键应用AI建议的修改
  - **多轮对话**：支持连续对话迭代优化内容

- **修改管理**：
  - **撤销/重做**：支持撤销每次AI修改操作
  - **修改历史**：查看页面的所有修改记录
  - **版本对比**：对比不同版本的页面差异

**使用流程**：

2. 在左侧页面导航中选择要编辑的页面

4. 在右侧对话面板输入修改指令（如"把标题改成'产品核心优势'"）

6. 确认修改或继续对话调整

## 14. 风险和挑战

### 14.1 技术风险

#### 风险1：页面解析和向量化准确性
**描述**：PPT页面内容解析可能不完整，向量化可能导致语义信息丢失
**应对措施**：
- 使用成熟的PPT解析库（python-pptx）
- 结合OCR技术提取图片中的文字
- 使用高质量的中文语义模型（m3e-base/bge-base-zh）
- 建立解析质量检测机制
- 支持用户手动补充页面标签和描述

#### 风险2：页面检索匹配度不足
**描述**：根据章节概要检索到的页面可能与用户期望不符
**应对措施**：
- 优化语义检索算法，结合关键词和向量相似度
- 增加备选页面数量，给用户更多选择
- 支持用户对检索结果进行反馈（标记相关/不相关）
- 允许用户手动搜索文档库补充页面
- 建立页面标签系统，提高检索准确性

#### 风险3：页面复制后的格式兼容性
**描述**：从不同来源PPT复制的页面可能存在格式不兼容或风格不一致
**应对措施**：
- 复制页面时保留原始格式信息
- 提供统一风格应用功能（可选）
- 预览时标注可能存在的格式问题
- 支持导出前统一调整字体和配色

#### 风险3：性能瓶颈
**描述**：大量文档和AI计算可能导致性能问题
**应对措施**：
- 使用缓存机制
- 异步处理耗时任务
- 水平扩展服务
- 优化数据库查询

### 14.2 业务风险

#### 风险1：用户章节定义不清晰
**描述**：用户可能无法准确描述章节需求，导致检索结果偏差
**应对措施**：
- 提供章节模板库供参考选择
- 提供章节描述示例和引导
- 支持用户从已有PPT导入章节结构
- 允许在组装过程中调整章节定义

#### 风险2：数据安全和隐私
**描述**：用户上传的PPT可能包含敏感信息
**应对措施**：
- 数据加密存储
- 实施访问控制
- 提供数据删除功能
- 本地部署确保数据不出境

##### 1. python-pptx ⭐ 核心推荐

**语言**：Python

**功能**：读取、创建、修改 PPTX 文件

**特点**：
- 可以提取文本、表格、图表、图片
- 获取幻灯片布局、形状、样式信息
- 支持创建和修改 PPT
- 开源免费，社区活跃

**适用场景**：本项目核心解析库

**GitHub**: https://github.com/scanny/python-pptx

**文档**: https://python-pptx.readthedocs.io/

**基础使用示例**：
```python
from pptx import Presentation
from pptx.util import Pt, RGBColor

# 读取PPT文件
prs = Presentation('example.pptx')

# 遍历幻灯片
for slide in prs.slides:
    # 遍历幻灯片中的形状
    for shape in slide.shapes:
        if shape.has_text_frame:
            print(shape.text)
        elif shape.has_table:
            # 提取表格内容
            table = shape.table
            for row in table.rows:
                for cell in row.cells:
                    print(cell.text)

# 提取样式信息
for shape in slide.shapes:
    if shape.has_text_frame:
        for paragraph in shape.text_frame.paragraphs:
            for run in paragraph.runs:
                font = run.font
                if font.name:
                    print(f"字体: {font.name}")
                if font.size:
                    print(f"字号: {font.size}")
                if font.color and font.color.rgb:
                    print(f"颜色: {font.color.rgb}")
```

#### 16.5.4 PPT 风格提取与应用

##### 1. python-pptx 样式提取

**功能**：分析 PPT 的配色、字体、布局

**示例代码**：
```python
from pptx import Presentation
from pptx.util import Pt, RGBColor
import json

def extract_ppt_styles(ppt_path):
    """提取PPT样式信息"""
    prs = Presentation(ppt_path)

    styles = {
        "color_scheme": {
            "primary": None,
            "secondary": [],
            "background": []
        },
        "fonts": {
            "title_fonts": set(),
            "body_fonts": set()
        },
        "layouts": []
    }

    # 提取配色方案
    theme = prs.slide_master.theme
    color_scheme = theme.color_scheme

    styles["color_scheme"]["primary"] = str(color_scheme.accents[0].rgb)

    # 遍历幻灯片提取样式
    for slide in prs.slides:
        layout = slide.slide_layout.name
        if layout not in styles["layouts"]:
            styles["layouts"].append(layout)

        for shape in slide.shapes:
            if shape.has_text_frame:
                for paragraph in shape.text_frame.paragraphs:
                    for run in paragraph.runs:
                        font = run.font
                        if font.name:
                            if paragraph.level == 0:
                                styles["fonts"]["title_fonts"].add(font.name)
                            else:
                                styles["fonts"]["body_fonts"].add(font.name)
                        if font.color and font.color.rgb:
                            color = str(font.color.rgb)
                            if color not in styles["color_scheme"]["secondary"]:
                                styles["color_scheme"]["secondary"].append(color)

    return styles

# 使用示例
styles = extract_ppt_styles("template.pptx")
print(json.dumps(styles, indent=2, ensure_ascii=False))
```

```python
from pymilvus import Collection, connections, FieldSchema, CollectionSchema, DataType, utility

# 连接Milvus
connections.connect(host="localhost", port="19530")

# 定义集合Schema
fields = [
    FieldSchema(name="id", dtype=DataType.VARCHAR, is_primary=True, max_length=36),
    FieldSchema(name="slide_id", dtype=DataType.VARCHAR, max_length=36),
    FieldSchema(name="content", dtype=DataType.VARCHAR, max_length=65535),
    FieldSchema(name="embedding", dtype=DataType.FLOAT_VECTOR, dim=768)
]
schema = CollectionSchema(fields, description="PPT Slide Embeddings")

# 创建集合
if not utility.has_collection("ppt_slides"):
    collection = Collection("ppt_slides", schema)

# 插入数据
data = [
    [str(uuid.uuid4()) for _ in range(100)],  # id
    slide_ids,  # slide_id
    contents,   # content
    embeddings  # embedding (768维向量)
]
collection.insert(data)
collection.flush()

# 创建索引
index_params = {
    "index_type": "IVF_FLAT",
    "metric_type": "COSINE",
    "params": {"nlist": 128}
}
collection.create_index("embedding", index_params)

# 查询
collection.load()
```

**功能**：图文多模态向量化

**使用示例**：
```python
import torch
import clip
from PIL import Image

# 加载模型
device = "cuda" if torch.cuda.is_available() else "cpu"

# 图像编码

    image_features = model.encode_image(image)

# 文本编码

with torch.no_grad():
    text_features = model.encode_text(text)

# 计算相似度
logits_per_image, logits_per_text = model(image, text)
similarity = logits_per_image.softmax(dim=-1).cpu().numpy()
```
